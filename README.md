# Лабораторна робота: Розгортання MQTT-брокера

## Про що лабораторна робота

Мета цієї лабораторної роботи — отримати практичні навички роботи з IoT-протоколом **MQTT**.  
У ході виконання роботи я розгорнув власний MQTT-брокер **Eclipse Mosquitto** за допомогою Docker, налаштував конфігурацію та перевірив обмін повідомленнями за принципом **Publish / Subscribe**.

---

## Структура проєкту

```text
stt-pz-3/
├── broker/
│   ├── mosquitto.conf       # Налаштування брокера (порт 1883, анонімний доступ)
│   └── docker-compose.yml   # Docker Compose для запуску Mosquitto
├── screenshots/             # Скріншоти, що підтверджують роботу
│   └── pub-sub.png
├── .editorconfig
├── .gitignore
└── README.md


Як запустити
Перейдіть до папки з конфігурацією брокера та запустіть контейнер:

cd broker
docker compose up -d
Брокер буде доступний за адресою:

mqtt://localhost:1883


Основні поняття MQTT
Topic (топік)
Ієрархічна адреса, наприклад sensor/temperature.
Брокер маршрутизує повідомлення між клієнтами за топіками.

Publish (публікація)
Надсилання повідомлення клієнтом у певний топік брокера.

Subscribe (підписка)
Клієнт підписується на топік і отримує всі нові повідомлення, які надходять у нього.

QoS (Quality of Service)
Рівень	Назва	Опис
QoS 0	At most once	Відправив і забув, без гарантій
QoS 1	At least once	Гарантована доставка, можливі дублікати
QoS 2	Exactly once	Рівно один раз, найнадійніше

Висновок
У результаті виконання лабораторної роботи я навчився розгортати MQTT-брокер за допомогою Docker, працювати з конфігурацією Mosquitto та перевіряти обмін повідомленнями між клієнтами. Отримані знання можна використовувати для подальшої роботи з IoT-системами.
